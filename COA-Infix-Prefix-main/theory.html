Infix, Postfix, and Prefix Notations

Introduction

In computer science and mathematics, there are three common ways to write mathematical expressions:

  • Infix: Operators are written between operands (e.g., A + B)
  • Prefix (Polish Notation): Operators precede their operands (e.g., + A B)
  • Postfix (Reverse Polish Notation): Operators follow their operands (e.g., A B +)

Infix Notation

Infix notation is the most common way of writing expressions where the operator is placed between its operands. While this is intuitive for humans, it requires additional processing for computers due to operator precedence and parentheses.

Examples:

  • A + B
  • (A + B) * C
  • A + B * C - D / E

Postfix Notation

In postfix notation, the operator follows its operands. This notation is particularly useful for stack-based evaluation and is the format used by many calculators.

Examples (with equivalent infix):

  • A B + (A + B)
  • A B + C * ((A + B) * C)
  • A B C * + D E / - (A + B * C - D / E)

Advantages of Postfix

  • No need for parentheses to dictate order of operations
  • Easier to evaluate using a stack
  • No operator precedence rules to consider during evaluation

Prefix Notation

Prefix notation places the operator before its operands. Like postfix, it doesn't require parentheses to dictate the order of operations.

Examples (with equivalent infix):

  • + A B (A + B)
  • * + A B C ((A + B) * C)
  • - + A * B C / D E (A + B * C - D / E)

Conversion from Infix to Postfix

The algorithm to convert infix expressions to postfix uses a stack to keep track of operators. Here's how it works:

  1. Initialize an empty stack and an empty output list
  2. Scan the infix expression from left to right
  3. If the current token is an operand, add it to the output
  4. If the current token is '(', push it onto the stack
  5. If the current token is ')', pop from the stack until '(' is encountered
  6. If the current token is an operator:
    • While there is an operator on top of the stack with higher or equal precedence, pop it to the output
    • Push the current operator onto the stack
  7. After processing all tokens, pop all remaining operators from the stack to the output

Example: Convert A + B * C to Postfix

  1. A → Output: A
  2. + → Push to stack: [+], Output: A
  3. B → Output: A B
  4. * → Higher precedence than +, push to stack: [*, +], Output: A B
  5. C → Output: A B C
  6. End of input, pop all: Output: A B C * +

Evaluating Postfix Expressions

Postfix expressions can be evaluated using a stack with the following algorithm:

  1. Initialize an empty stack
  2. Scan the postfix expression from left to right
  3. If the current token is an operand, push it onto the stack
  4. If the current token is an operator:
    • Pop the top two operands from the stack (the second popped is the left operand)
    • Apply the operator to the operands
    • Push the result back onto the stack
  5. After processing all tokens, the stack should contain exactly one element: the result

Example: Evaluate 3 4 2 * + 5 -

  1. 3 → Stack: [3]
  2. 4 → Stack: [3, 4]
  3. 2 → Stack: [3, 4, 2]
  4. * → Pop 4 and 2, compute 4*2=8 → Stack: [3, 8]
  5. + → Pop 3 and 8, compute 3+8=11 → Stack: [11]
  6. 5 → Stack: [11, 5]
  7. - → Pop 11 and 5, compute 11-5=6 → Stack: [6]
  8. Result: 6

Operator Precedence and Associativity

When converting from infix to postfix, operator precedence and associativity determine the order of operations:

Operator Precedence Associativity
^ (exponentiation) 4 Right to left
*, /, % 3 Left to right
+, - 2 Left to right

Note:

Parentheses have the highest precedence and can be used to override the default precedence rules.
